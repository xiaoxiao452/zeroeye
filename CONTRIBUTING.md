# Contributing to ZeroEye

Thank you for your interest in contributing to ZeroEye! This document provides guidelines and instructions for contributing.

## Development Setup

### Prerequisites
- Node.js 18+
- npm or yarn

### Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/your-username/zeroeye.git
   cd zeroeye
   ```

3. Install dependencies:
   ```bash
   npm install
   ```

4. Create a branch for your changes:
   ```bash
   git checkout -b feat/your-feature-name
   ```

## Build Instructions

```bash
# Install dependencies
npm install

# Run the build
npm run build

# Run tests
npm test

# Start development server
npm run dev
```

## Code Style

- Follow the existing code style in the project
- Use 2-space indentation
- Write meaningful commit messages following conventional commits format
- Run the linter before submitting: `npm run lint`

## Pull Request Process

1. Ensure your code builds and passes tests
2. Update documentation if needed
3. Submit a PR with a clear description of what you changed and why
4. Link any related issues in your PR description

## Bounty Process

ZeroEye participates in the community bounty program. If you're working on a bounty issue:

1. Comment `/attempt` on the issue you want to work on
2. Wait for assignment
3. Submit your PR
4. The maintainer will review and process the bounty

## Questions?

Open an issue or reach out to the maintainers.
